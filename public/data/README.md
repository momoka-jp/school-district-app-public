# public/data

## 概要

本ディレクトリは，本アプリケーションが参照する **入力データおよび生成データ** の置き場である．  
`public/data/config.json` を中心に，データのパス解決や出力先が管理される．  

> ⚠️ **Important**
>
> 本ディレクトリには実データではなく **合成（サンプル）データ**（定員，児童・生徒数）を用いている．  
> 実在の自治体の数値や意思決定を直接反映するものではない．

## 📂 ディレクトリ構成
```text
public/data/
├─ p29/                 # 奈良県のオープンデータ（学校・学校区・町丁目）
├─ output/              # 前処理・マージ後の生成データ
├─ city_codes.csv       # 市町村コード一覧
├─ config.json          # データ生成・読み込み設定
└─ README.md            # 本ファイル
```

- `config.json`  
  - アプリとスクリプトで共通に参照される設定ファイル  
  - (自治体名，各種ファイル名，EPSG などを定義)  
  - `app/api/*` および `scripts/*` から参照される
- `city_codes.csv`  
  - 都道府県名・市区町村名から行政コードを引くためのマスターデータ
- `output/`  
  - 生成データの保存先（前処理・マージ後のデータは原則ここに集約）
- `p29/` など  
  - 自治体別の原データ／中間データ  
  - 例：`p29/29209/...`
  - 初期状態は **奈良県（都道府県コード 29）** のみ

  **p29 ディレクトリの内訳（奈良県）**

  p29/ は，本リポジトリで **初期状態として提供されている唯一の都道府県データ**であり，
  サンプル実行用として奈良県（29）・生駒市（29209）を含んでいる．

  - `p29/A27-23_29_GML/`  
    奈良県の 小学校区ポリゴン（国土数値情報 A27）
  - `p29/A32-23_29_GML/`  
    奈良県の 中学校区ポリゴン（国土数値情報 A32）
  - `p29/P29-21_29_GML/`  
    奈良県の 学校位置・属性データ（国土数値情報 P29）
  - `p29/29209/`  
    生駒市（市区町村コード 29209）に関する中間データ例

## よく使う生成物（output）

- `schools_base.geojson`  
  学校位置データのベース    （`scripts/generate_schools_base.py`）
- `merged.geojson`  
  町丁目ポリゴンと学校区ポリゴンの空間結合結果    （`scripts/create_merged_geojson.py`）
- `merged_with_students.geojson`  
  町丁目ごとに年度別の児童・生徒数を付与した GeoJSON    （`scripts/merged_students_multi_year.py`）
- `schools.geojson`  
  学校点に定員情報を統合したデータ    （`app/api/merge-schools`）
- `schools.original.geojson`  
  学校データのリセット用原本
- `available_years.json`  
  取り扱い可能な年度一覧    （`scripts/merged_students_multi_year.py`）
- `adj.json`  
  町丁目間の隣接関係    （`/api/extract-adjacency` の結果）

## オープンデータについて

### 学校データ（位置・属性）

- 国土数値情報 学校データ（P29）
  - 出典：  
    https://nlftp.mlit.go.jp/ksj/gml/datalist/KsjTmplt-P29-v2_0.html
  - 含まれる主な属性：
    - 緯度・経度
    - 市町村コード
    - 学校コード
    - 学校分類（小学校・中学校 等）
    - 学校名
    - 住所（文字列）
    - 設置区分（市区町村立・私立 等）
    - 休校コード（0:調査なし, 1:開校中, 9:休校中）

#### 学校区ポリゴン

- 小学校区（A27）：  
  https://nlftp.mlit.go.jp/ksj/gml/datalist/KsjTmplt-A27-2023.html
- 中学校区（A32）：  
  https://nlftp.mlit.go.jp/ksj/gml/datalist/KsjTmplt-A32-2023.html

#### 町丁目ポリゴン

- 国勢調査 町丁・字等別境界データ  
  https://geoshape.ex.nii.ac.jp/ka/resource/
  Town-level boundary data:© CODH, based on “2020 Population Census Town-Level Boundary Data” (e-Stat), CC BY 4.0.

### Data Processing Notes
- 通学距離は町丁目ポリゴンの重心と学校位置の 直線距離として計算している
- 重心計算には geopandas を使用している

#### 実データについて

以下のデータを用意する．

##### (1) `school_capacity.json`

- 各学校の **受入可能人数（定員）** を定義する JSON ファイルである．
- 最適化モデルにおける **学校規模制約** や，定員超過／未充足の評価に用いられる．

**形式**
```json
{
  "学校名A": {
    "min": 最小人数,
    "max": 最大人数
  },
  "学校名B": {
    "min": 最小人数,
    "max": 最大人数
  }
}
```

**記述例**
```json
{
  "A小学校": {
    "min": 150,
    "max": 600
  },
  "B中学校": {
    "min": 200,
    "max": 900
  },
  "C小学校": {
    "min": 150,
    "max": 600
  }
}
```

**各項目の意味**

- キー（例: "A小学校"）
  - 学校名
  - schools_base.geojson に含まれる学校名と 完全一致している必要がある
- min
  - 当該学校の最小許容人数
  - 学校規模が極端に小さくなることを防ぐための下限
- max
  - 当該学校の最大許容人数（定員）
  - 定員超過を防ぐための上限

##### (2) `小中町丁目別学校区別学齢別性別集計.csv`

- 町丁目単位で集計された**年度別の児童・生徒数**を記述したCSVファイル
- 最適化モデルにおける **需要データ** および，UI 上での **人口分布・年度別変化の可視化** に用いられる．

本ファイルは，**Excel 等で作成した表を CSV 形式で保存したもの**を想定している．  
アプリケーションは CSV の列構成を直接読み取り，
利用可能な年度を自動的に判別するため，
あらかじめ全年度を固定する必要はない．

---

**想定される作成方法（参考）**

- 自治体が保有する集計データを Excel 等で整理
- 必要な列を満たした状態で CSV 形式として保存
- 保存した CSV をそのまま入力データとして利用

**参考：Excel での作成例**

本 CSV ファイルは，**Excelで作成した表をそのまま CSV として保存したもの**を入力として利用する．
CSV 固有の仕様を理解する必要はなく，Excel 上の列構成が重要である．

**行構成**

- 1 行目：系列名（データの種類）
- 2 行目：列名（識別情報・年度）
- 3 行目以降：町丁目ごとの実データ

**列構成（概要）**

（例）
|         |          |      |                | 就学率    |          | 児童学齢住民数 | … | 生徒学齢住民数 | … |児童数 | … | 生徒数 | … |
|---------|----------|------|----------------|----------|----------|--------------|---|----------|---|----------|---|----------|---|
| 小学校区 | 中学校区   | 町名  | 町丁目名        | 小学校区  | 中学校区   | 2025年度     | … |2025年度 | … |2025年度 | … |2025年度 | … |
| A小学校  | B中学校   | C    | A小学校-B中学校-C| 90%      | 97%       | 30          | … | 20 | … |27 | … | 29 | … |

**列の意味**

- 1〜6 列目：識別情報＋就学率
  -  1.小学校区
  -  2.中学校区
  -  3.町名
  -  4.町丁目名
  -  5.小学校区就学率(%)
  -  6.中学校区就学率(%)
- 7 列目以降：年度
  - `2025年度`, `2026年度`, `2027年度`, …
  - 年度数・対象期間は任意
  - 列名に含まれる年度表記を，アプリケーションが自動的に認識する

**CSV への変換について**

- Excel で作成した表について，行・列構成を変更せずに CSV 形式で保存すればよい
- エクセルの作成の仕方は`public/data/小中町丁目別学校区別学齢別性別集計_sample.csv`をダウンロードの上，そこに記載してある手順に則って作成すればよい

**データの考え方**

- 児童学齢住民数 / 生徒学齢住民数  
  - 町丁目内に居住する学齢人口  
  - 自治体が保有する集計データを想定する
- 就学率  
  - 当該町丁目の学齢住民のうち，
    該当学校区に就学する割合
- 児童数 / 生徒数  
  - 以下の計算により算出される需要人数：

    児童数 ＝ 児童学齢住民数 × 小学校区就学率  
    生徒数 ＝ 生徒学齢住民数 × 中学校区就学率
  
  アプリケーションでは，

  - 児童数 → 小学校需要
  - 生徒数 → 中学校需要

  として扱われる．
